# 编码规范和工具

## 命名规范
* 文件的命名：
  - 路由：.vue文件、文件夹 使用横线连接 (kebab-case)
  - 组件：所有的Component文件都是以大写开头 (PascalCase)
  - js：.js文件都遵循横线连接 (kebab-case)
  
* .vue 文件事件方法的命名
  - 事件相关的函数：on + 触发事件的元素名称 + 事件名称
  ```js
    onSaveBtnClick() {}
    onDeleteBtnClick() {}
    onAddBtnClick() {}
    onChainNameInputBlur() {}
  ```


## 工具使用及配置
* EditorConfig: 跨编辑器和IDE编写代码，保持一致的简单编码风格；
  - EditorConfig项目包含一个用于定义编码样式的文件格式和一个文本编辑器插件集合，这些文本编辑器插件使编辑器可以读取文件格式并遵循定义的样式。
  - 有助于从事同一项目的多个开发人员在跨多个编辑器和IDE使用时保持一致的编码风格
  - 常用的：Webstorm、IntelliJ IDEA等 已经支持EditorConfig，vscode 需安装 EditorConfig for VS Code
  - [editorConfig](./.editorconfig)

* Prettier: 专注于代码格式化的工具，美化代码；
  - 它在美化代码方面有很大的优势，配合ESLint可以对ESLint格式化基础上做一个很好的补充
  - vscode 安装 Prettier - Code formatter 设为默认格式化器（也可以根据不同类型的文件设置不同的格式化器）
  - 配置:
    ```json
      {
      // 设置语句末尾不加分号
      "semi": false,
      // 设置强制单引号
      "singleQuote": true,
      // 为多行数组的非末尾行添加逗号
      "trailingComma": all,
      // 每行最大宽度 100
      "printWidth": 100,
      // jsx中使用单引号
      "jsxSingleQuote": true
    }

  ```

* ESLint：作代码质量检测、编码风格约束等；
  - 检查语法错误，避免低级bug
  - 统一团队代码风格
  - 确保代码遵循最佳实践
  - vscode settings.json 配置
    ```json
      // 保存自动修复所有，
      "editor.codeActionsOnSave": {
          "source.fixAll": true
      }
    ```
* 如何解决Prettier与ESLint的配置冲突问题？
  > 代码格式化时采用Perttier规则，而代码校验使用的是ESLint，同一个规则配置不一致，就会出现冲突问题；
  - 修改任意一个配置文件，让两者规则一致
